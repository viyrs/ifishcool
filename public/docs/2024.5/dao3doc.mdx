如果用过神奇代码岛的用户都知道，官方写的编辑器教程是在语雀平台上的，尤其是 API，那非常的简约，甚至不统一的写法，有些地方为了赶工，苦涩难懂，后续直接断更了。这让用户很难阅读。

当我自己在用这些文档开发和回答问题时，明显感觉到几个问题：

- 不同页面的写法风格不统一，同一个概念在不同地方用不同名字；
- 代码示例偏少，很多 API 只有一句话描述，没有结合场景；
- 断更之后，新功能、新最佳实践都散落在社区帖子和群聊里，很难系统化。

于是，我决定重新从 0 开始整理官方文档，结合目前的 AI 辅助，整理成统一的帮助文档。

## 技术栈选择：为什么是 VitePress

首先，技术栈选择 VitePress，因为它特别好用：

- 本地编写 MD 文件，保存后即可动态刷新，**即写即所得**，非常适合写文档；
- 默认支持侧边栏、搜索、版本化等文档常用功能，省去了大量“造轮子”的时间；
- 后续可以直接部署在官方平台或 git 仓库，天然支持**文档共建**：提 PR、提 Issue 的流程都很成熟。

![](https://cdn-community.bcmcdn.com/47/community/RIFDckzLS6Nlk2Tk9wUSQTt7OssnBMD1tXl4UzeCZXcl.png?hash=FqLxs6YgYp3bdqsRNv9WeCuYuYYy)

在这个基础上，我按照“产品 → 功能 → API → 示例”的结构，逐步把原来散落在语雀和社区里的内容，搬运、清洗、重写到一个统一的文档站里。

## 加入示例：让用户一眼看到“能做什么”

里面还结合了自己和平台优秀的作品当作示例，让新用户一眼就爱上他。哇，原来能实现这么棒的效果。

![](https://static.codemao.cn/pickduck/HkrmRKlGkl.gif?hash=lm78J0LS8CUIYMHZDEuyLx-7CMLQ)

这些示例的作用主要有两点：

- 对完全没接触过神岛的新用户来说，先看到“最终效果”，再去看“怎么实现”，学习动机会强很多；
- 对已经有基础的创作者来说，可以把示例当作**可运行的模板**，复制、修改，比从空白开始高效得多。

## 与 AI 的联动：写给人，也写给机器看

目前，这个文档有产品使用手册，还有 API 手册，可以说应有尽有。同时写好了，喂给 AI，AI 整合的效率大大提高。

![](https://cdn-community.bcmcdn.com/47/community/eSlGMrK2HV8HS9Ce6SNEJVWnTpuLQKMb8ZM0J8t09WNR.png?hash=Fj_wvWpMoYweO-fDm58wUoNUi4-g)

一份结构清晰、命名统一的文档，对 AI 来说也更友好：

- 更容易按“功能 / 模块”切分成语义块；
- 同一个概念不会出现十几种叫法，减少了检索歧义；
- 示例代码和说明紧挨着，便于 AI 在回答问题时引用完整上下文。

这也为我 later 在神岛平台上接入 AI 助手打下了基础——当底层文档健康时，AI 的表现也会更稳定、可靠。
